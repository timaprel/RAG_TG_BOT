# RAG_TG_BOT

Telegram-бот для хранения фактов и генерации ответов с использованием технологии RAG (Retrieval-Augmented Generation) на основе LLM модели Saiga 8B и векторной базы данных ChromaDB.

## О проекте

Бот позволяет:
- Сохранять текстовые факты в векторную базу данных
- Искать наиболее релевантные факты по запросу
- Генерировать ответы с помощью языковой модели Saiga Llama3 8B
- Использовать RAG-подход: сначала искать информацию в базе, затем генерировать ответ на её основе

Проект демонстрирует применение RAG в реальном приложении, объединяя поиск информации и генеративные возможности LLM.

## Возможности

| Команда | Описание |
|---------|----------|
| `/start` | Приветственное сообщение |
| `/add <текст>` | Добавить новый факт в базу данных |
| `/get_all` | Показать все сохранённые факты |
| `/search <запрос>` | Найти наиболее похожий факт в базе |
| `/generate <промпт>` | Сгенерировать ответ только на основе LLM |
| `/rag <запрос>` | Найти факт и сгенерировать ответ на его основе |
| `/help` | Показать список всех команд |

## Технологии

- **Язык**: Python 3.10+
- **LLM**: [Saiga Llama3 8B](https://huggingface.co/IlyaGusev/saiga_llama3_8b) от Ильи Гусева
- **Векторная БД**: ChromaDB
- **Фреймворк для бота**: aiogram 3
- **Библиотеки**: transformers, torch, dotenv

# Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/yourusername/RAG_TG_BOT.git
cd RAG_TG_BOT
```

## 2. Установите зависимости
```bash
pip install -r requirements.txt
```
## 3. Скачайте модель
Запустите скрипт для загрузки модели Saiga Llama3 8B:

```bash
python download_model.py
```
Модель будет сохранена в кэш Hugging Face.

Путь по умолчанию: ```C:/Users/user/.cache/huggingface/hub/```.

## 4. Настройте токен бота

В файле .env введите код полученный от BotFather
```
BOT_KEY=ваш_токен_от_BotFather
```

## 5. Запустите бота

```bash
python run.py
```
