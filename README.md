# RAG_TG_BOT

Telegram-бот для хранения фактов и генерации ответов с использованием технологии RAG (Retrieval-Augmented Generation) на основе LLM модели Saiga 8B и векторной базы данных ChromaDB.

## О проекте

Бот позволяет:
- Сохранять текстовые факты в векторную базу данных
- Искать наиболее релевантные факты по запросу
- Генерировать ответы с помощью языковой модели Saiga Llama3 8B
- Использовать RAG-подход: сначала искать информацию в базе, затем генерировать ответ на её основе

Проект демонстрирует применение RAG в реальном приложении, объединяя поиск информации и генеративные возможности LLM.

## Возможности

| Команда | Описание |
|---------|----------|
| `/start` | Приветственное сообщение |
| `/add <текст>` | Добавить новый факт в базу данных |
| `/get_all` | Показать все сохранённые факты |
| `/search <запрос>` | Найти наиболее похожий факт в базе |
| `/generate <промпт>` | Сгенерировать ответ только на основе LLM |
| `/rag <запрос>` | Найти факт и сгенерировать ответ на его основе |
| `/help` | Показать список всех команд |

## Описание кода
Код запуска проекта:
run.py

```python
context = db.search_one(user_query)      # поиск
prompt = f"Контекст: {context}\nВопрос: {user_query}"
response = llm.generate(prompt)          # генерация
```
Код с конфигурацией базы данных:
db.py
```python
def search_one(self, query: str) -> str:
    res = self.collection.query(query_texts=[query], n_results=1)
    return res["documents"][0][0] if res["documents"] else ""
```
Код с конфигурацией работы LLM:
llm.py
```python
def generate(self, text: str, max_tokens: int = 250) -> str:
    prompt = f"Пользователь: {text}\nАссистент:"
    inputs = self.tokenizer(prompt, return_tensors="pt").to(self.device)
    output = self.model.generate(**inputs, max_new_tokens=max_tokens)
    return self.tokenizer.decode(output[0], skip_special_tokens=True)
```

## Вывод

| Критерий | Без RAG (/generate) | С RAG (/rag) |
| ------------- | ------------- | ------------- |
| Точность | Низкая (галлюцинации) | Высокая (ответ по фактам) |
| Источник | Общие знания модели | Ваша база данных |
| Контроль | Нельзя проверить | Можно (факт из БД) |
| Актуальность | Ограничена датой обучения | Всегда свежие факты |

RAG полезен, когда:  
* Нужны ответы на основе конкретных данных
* Важна проверяемость источника

На практике было замечено:
* Поиск не идеален - векторы ищут по смыслу, а не по ключевым словам
* Модель Saiga обучена на диалогах - может давать «двойной ответ» (развёрнутый + краткий)
* Для работы нужны вычислительные ресурсы: модель на 8B может запуститься на процессоре, но будет работать медленно.

# Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/yourusername/RAG_TG_BOT.git
cd RAG_TG_BOT
```

## 2. Установите зависимости
```bash
pip install -r requirements.txt
```
## 3. Скачайте модель
Запустите скрипт для загрузки модели Saiga Llama3 8B:

```bash
python download_model.py
```
Модель будет сохранена в кэш Hugging Face.

Путь по умолчанию: ```C:/Users/user/.cache/huggingface/hub/```.

## 4. Настройте токен бота

В файле .env введите код полученный от BotFather
```
BOT_KEY=ваш_токен_от_BotFather
```

## 5. Запустите бота

```bash
python run.py
```
